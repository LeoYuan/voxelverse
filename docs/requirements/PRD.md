# 《体素世界》（VoxelVerse）产品需求文档 v1.0

> 状态：已沉淀 | 最后更新：2026-04-21

---

## 1. 产品定位与核心循环

### 1.1 一句话描述
一个能在浏览器里运行的、带图灵完备红石电路系统的轻量体素沙盒游戏。

### 1.2 双核心循环

| 循环 | 体验 | 解锁阶段 |
|---|---|---|
| **创造循环** | 无限资源 + 飞行 → 建造任意结构 | Phase 1-2 |
| **生存循环** | 采集 → 合成 → 战斗/躲避 → 发展科技 → 征服世界 | Phase 3 |
| **工程循环** | 挖矿 → 获得红石 → 设计电路 → 自动化/计算 | Phase 4-5 |

## 2. 功能性需求（FR）

### A. 世界系统
- **FR-W1**：无限（伪无限）程序生成的 3D 体素世界，最小单位 1×1×1 方块
- **FR-W2**：基于 Chunk（16×16×16）的流式加载与卸载
- **FR-W3**：至少 3 个生物群系：平原、森林、山地
- **FR-W4**：昼夜循环（20 分钟一个游戏日），光照影响怪物生成
- **FR-W5**：地下洞穴生成（简单 3D Perlin 噪声挖空）

### B. 玩家系统
- **FR-P1**：第一人称/第三人称可切换视角
- **FR-P2**：WASD 移动、空格跳跃、Shift 潜行、鼠标视角
- **FR-P3**：生存模式属性：生命值（20 点）、饥饿值（20 点）、护甲值
- **FR-P4**：背包系统：40 格存储（4×10），支持物品堆叠（最大 64）
- **FR-P5**：合成系统：2×2 随身合成 + 3×3 工作台合成
- **FR-P6**：工具系统：木/石/铁/金/钻石五级，不同挖掘速度和耐久

### C. 方块系统
- **FR-B1**：方块类型配置化，至少包含：
  - 自然类：草方块、泥土、石头、沙子、砂砾、矿石（煤、铁、金、红石、钻石）
  - 木质类：原木、木板、木棍
  - 装饰类：玻璃、楼梯、半砖
  - 功能类：工作台、熔炉、箱子、门、梯子
  - 红石类：红石粉、红石火把、红石块、拉杆、按钮、压力板、红石灯、中继器、比较器、活塞
- **FR-B2**：方块状态系统：朝向、开启/关闭、充能等级（0-15）、延迟等
- **FR-B3**：物理：掉落物实体（受重力）、简单 AABB 碰撞

### D. 生存与战斗系统
- **FR-S1**：饥饿消耗：跑步/跳跃/受伤恢复消耗饥饿
- **FR-S2**：食物系统：苹果、面包、生/熟肉（至少 5 种食物）
- **FR-S3**：敌对生物：僵尸（近战）、骷髅（远程）、爬行者（自爆）
- **FR-S4**：生物 AI：简单寻路、仇恨范围（16 格）、昼夜行为
- **FR-S5**：死亡与重生：死亡掉落物品，重生点设置（床）

### E. 红石与自动化系统
- **FR-R1**：红石信号系统：信号强度 0-15，每格红石粉衰减 1
- **FR-R2**：逻辑门实现：非门（火把）、与门/或门（可由布线自然实现）
- **FR-R3**：时序控制：中继器（1-4 tick 延迟，信号放大）
- **FR-R4**：比较器：检测容器满度、减法模式（可选）
- **FR-R5**：执行器：活塞、发射器/投掷器（简化版）、红石灯
- **FR-R6**：输入设备：拉杆、按钮、压力板、阳光传感器（简化版）
- **FR-R7**：存储单元：RS 锁存器、T 触发器（玩家可用基础元件搭建）
- **FR-R8**：图灵完备验证：组合逻辑 + 时序逻辑 + 存储
- **FR-R9**：命令方块（简化版）：创造模式可用，输入预设指令改变环境

### F. UI 与交互系统
- **FR-U1**：HUD：准星、生命值条、饥饿值条、手持物品显示
- **FR-U2**：物品栏：快捷栏（9 格）、背包、合成界面
- **FR-U3**：暂停菜单：返回游戏、保存、设置、退出
- **FR-U4**：调试界面（F3 风格）：显示 FPS、坐标、朝向、Biome、光照等级

## 3. 非功能性需求（NFR）

- **NFR-1 性能**：目标 60 FPS，渲染距离 8 Chunks（128×128 格），同屏方块数 > 10 万
- **NFR-2 平台**：现代桌面浏览器，键鼠操作，未来预留触屏接口
- **NFR-3 存档**：世界序列化至 IndexedDB，支持多存档槽位（≥3 个）
- **NFR-4 构建**：Vite 热更新，单页应用，可部署至 Vercel/Netlify
- **NFR-5 代码**：TypeScript 严格模式，模块化架构，单元测试覆盖红石逻辑引擎

## 4. 优先级矩阵

| 优先级 | 说明 |
|---|---|
| **P0 - 必须有** | 体素渲染、基础交互、生存核心循环、红石基础电路、存档 |
| **P1 - 应该有** | 音效、生物 AI、熔炉、中继器、设置菜单 |
| **P2 - 可以有** | 流动水、比较器减法模式、成就系统、PWA |
| **P3 - 延后** | 多人联机、下界/末地、复杂生物、模组 API |

---

## 5. 风险与应对

| 风险 | 概率 | 应对策略 |
|---|---|---|
| 红石时序 Bug | 高 | 双缓冲状态机，每 tick 只读 current，只写 next，tick 结束 swap |
| 性能崩溃 | 中 | 早期加入视锥体剔除；红石电路只激活区域计算 |
| 需求蔓延 | 高 | 严守 P0 清单 |
| 存档损坏 | 中 | 保存时写双份（主存档 + .backup） |
